# Инструкция по запуску и тестированию

Документ описывает запуск retrieval-слоя проекта `vector-db-rag` на локальной машине.

## 1. Требования

- Python `3.11+`.
- Docker Desktop или совместимый Docker runtime.
- Доступ к OpenAI API для dense indexing и dense/hybrid query.
- PowerShell для команд ниже.

## 2. Установка зависимостей

Из корня проекта:

```powershell
python -m pip install -e .
```

Проверка CLI:

```powershell
vector-db-rag --help
```

Если пакет не установлен, временно можно запускать через `PYTHONPATH`:

```powershell
$env:PYTHONPATH = "src"
python -m vector_db_rag --help
```

## 3. Переменные окружения

Создать `.env`:

```powershell
Copy-Item .env.example .env
```

Заполнить:

```text
OPENAI_API_KEY=...
QDRANT_URL=http://localhost:6333
QDRANT_API_KEY=
```

Для локального Qdrant `QDRANT_API_KEY` обычно пустой.

## 4. Запуск Qdrant

```powershell
docker compose up -d qdrant
```

Проверка:

```powershell
vector-db-rag check-qdrant
```

Остановка:

```powershell
docker compose down
```

## 5. Валидация корпуса чанков

```powershell
vector-db-rag validate-chunks
```

Ожидаемые ключевые значения для текущего корпуса:

- `chunks`: `138`;
- `articles`: `12`;
- `chunker_version`: `1.2.0`;
- `config_hash`: `0623918e2d7b`.

## 6. Создание коллекции Qdrant

Первичное создание:

```powershell
vector-db-rag create-collection
```

Полное пересоздание:

```powershell
vector-db-rag create-collection --recreate --yes
```

Коллекция:

- имя: `chunks`;
- dense vector: `dense_retrieval`, `3072`, `Cosine`;
- sparse vector: `sparse_bm25`;
- payload indexes: `chunk_id`, `article_id`, `article_meta.year`, `article_meta.arxiv_id`, `section`, `section_hierarchy`, `chunk_type`, `question_types`, `keywords`, `index`.

## 7. Индексация

### Dense indexing

Требует `OPENAI_API_KEY`.

```powershell
vector-db-rag index-chunks --dense
```

Что делает команда:

1. Валидирует `chunks 2/docs/dic_chunks.json`.
2. Строит `retrieval_text`.
3. Получает embeddings через OpenAI `text-embedding-3-large`.
4. Загружает dense-вектор `dense_retrieval` и payload в Qdrant.
5. Кэширует embeddings в `data/cache/embeddings`.
6. Пишет manifest в `data/index_manifest.json`.

### Sparse indexing

```powershell
vector-db-rag index-chunks --sparse
```

Что делает команда:

1. Строит `sparse_text`.
2. Обучает локальный BM25 encoder на текущем корпусе.
3. Сохраняет encoder в `data/cache/sparse_bm25.json`.
4. Обновляет sparse-вектор `sparse_bm25` в Qdrant.
5. Добавляет в payload `sparse_text` и sparse metadata.

### Полная переиндексация

```powershell
vector-db-rag create-collection --recreate --yes
vector-db-rag index-chunks --dense --sparse
```

## 8. Поиск

### Dense-only

```powershell
vector-db-rag query "methods based on GPT-2 for time series forecasting" --profile dense_only --limit 5
```

### Sparse-only

Не вызывает OpenAI API, но требует `data/cache/sparse_bm25.json`.

```powershell
vector-db-rag query "GPT-2 MSE ETT" --profile sparse_only --limit 5
```

### Hybrid default

Требует dense embeddings и sparse BM25 cache.

```powershell
vector-db-rag query "methods based on GPT-2 for time series forecasting" --profile hybrid_default --limit 10
```

### Entity exact

```powershell
vector-db-rag query "methods based on GPT-2" --profile entity_exact --keyword GPT-2 --section Methodology --limit 5
```

### Table numeric

```powershell
vector-db-rag query "MSE results on ETT datasets" --profile table_numeric --limit 10
```

Если `--chunk-type` не задан, профиль `table_numeric` автоматически ограничивает поиск типами:

- `table`;
- `table_with_prose`;
- `formula_heavy`.

## 9. Фильтры поиска

Доступные CLI-фильтры:

```powershell
--article-id arxiv_2403_05798
--arxiv-id 2403.05798
--section Methodology
--chunk-type table
--question-type А
--keyword GPT-2
--keyword GPT-2,MSE,ETT
--year-gte 2023
--year-lte 2026
```

Фильтры передаются в Qdrant как payload filters.

## 10. Answer и Eval API

### Grounded answer

Локальный grounded answer без вызова LLM:

```powershell
vector-db-rag answer "Какие backbone и tuning strategies использует Time-LLM?" --profile sparse_only --limit 1 --article-id time-llm --max-context-chunks 1 --max-chars-per-chunk 900 --dry-run --json --include-prompts
```

### Подготовка Eval API payload

Dry-run для одного `question_id`:

```powershell
vector-db-rag eval-run --question-id time-llm_q06 --profile sparse_only --limit 1 --max-context-chunks 1 --dry-run --json
```

Команда `eval-run` собирает для каждого вопроса:

- `question_id`;
- `answer` на русском языке;
- `chunks` с verbatim-фрагментами статьи.

По умолчанию `eval-run` формирует русский grounded fallback-answer без вызова LLM. Если нужен ответ через DeepSeek, добавить `--with-llm` и определить `DEEPSEEK_API_KEY`.

### Отправка в Eval API

```powershell
$env:EVAL_API_KEY = "..."
vector-db-rag eval-run --run-id smoke-time-llm --article-id time-llm --profile sparse_only --limit 3 --max-context-chunks 3 --json
```

Для Eval API критично отправлять поле `chunks`, а не `citations`.

## 11. Запуск тестов

Базовый запуск:

```powershell
python -m pytest -p no:cacheprovider
```

Флаг `-p no:cacheprovider` отключает pytest cache. На текущей Windows-машине обычный pytest может создавать предупреждения `PytestCacheWarning` из-за отказа доступа к временным `pytest-cache-files-*`.

Проверка синтаксиса:

```powershell
python -m compileall src tests
```

Проверка CLI без внешней инфраструктуры:

```powershell
$env:PYTHONPATH = "src"
python -m vector_db_rag validate-chunks
python -m vector_db_rag query --help
python -m vector_db_rag index-chunks --help
```

## 12. Smoke-сценарий

Полный локальный сценарий:

```powershell
python -m pip install -e .
Copy-Item .env.example .env
# заполнить OPENAI_API_KEY в .env
docker compose up -d qdrant
vector-db-rag validate-chunks
vector-db-rag create-collection --recreate --yes
vector-db-rag index-chunks --dense --sparse
vector-db-rag query "GPT-2 MSE ETT" --profile hybrid_default --limit 10
vector-db-rag eval-run --question-id time-llm_q06 --profile sparse_only --limit 1 --max-context-chunks 1 --dry-run --json
```

## 13. Типовые проблемы

### `No module named vector_db_rag`

Пакет не установлен. Выполнить:

```powershell
python -m pip install -e .
```

или временно:

```powershell
$env:PYTHONPATH = "src"
```

### `OPENAI_API_KEY is not set`

Нужно заполнить `.env` или переменную окружения:

```powershell
$env:OPENAI_API_KEY = "..."
```

### `qdrant-client is not installed`

Установить зависимости:

```powershell
python -m pip install -e .
```

### Qdrant недоступен

Проверить контейнер:

```powershell
docker compose ps
docker compose logs qdrant
```

### Sparse encoder cache not found

Нужно выполнить:

```powershell
vector-db-rag index-chunks --sparse
```
