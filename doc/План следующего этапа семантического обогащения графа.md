# План следующего этапа семантического обогащения графа

**Статус:** Completed  
**Дата:** 2026-05-22  
**Область:** следующий этап после рабочего минимального графа `Article` / `Chunk` / `MENTIONS`  

## Текущее состояние

К моменту старта этого этапа уже реализованы и проверены:

- локальный Neo4j-контур рядом с Qdrant;
- графовая схема и идемпотентный импорт корпуса `chunks 2/docs/dic_chunks.json`;
- узлы `Article`, `Chunk` и базовые сущности первого слоя;
- связи `CONTAINS` и `MENTIONS`;
- сценарий `Qdrant-first + расширение графовым контекстом`.

Базовая метрика на старте этапа:

| Показатель | Значение |
| --- | --- |
| Статей | 12 |
| Чанков | 138 |
| `MENTIONS` | 832 |
| Неразобранных `keywords` | 320 |

Главное узкое место на старте этапа было не инфраструктурное, а семантическое: часть `keywords` оставалась вне онтологии, а несколько классов `graph-ontology/ontology.md` вообще не наполнялись при импорте.

## Цель этапа

Сделать граф существенно ближе к полной онтологии без перехода к хрупкому LLM-извлечению:

1. Расширить детерминированную классификацию `keywords -> ontology class`.
2. Начать наполнять дополнительные классы:
   - `TuningStrategy`;
   - `DataPreprocessing`;
   - `Pipeline`;
   - `Experiment`;
   - `ExperimentResult`.
3. Снизить число неразобранных `keywords`.
4. Сделать отчёт импорта пригодным для коротких итераций по нормализации.

## Выполненные работы

### Шаг 1. Расширено покрытие онтологии на этапе импорта

В модуль нормализации добавлены:

- точные правила для частых unresolved keywords;
- отдельные правила для training/tuning-сигналов;
- отдельные правила для preprocessing-сигналов;
- отдельные правила для pipeline и experiment-сигналов.

Фактический эффект:

- часть прежних `Technique` была переотнесена в более точные классы;
- значимая часть частотных unresolved keywords перестала теряться;
- граф стал ближе к целевой модели `ontology.md`.

### Шаг 2. Расширены графовая схема и импорт

Реализовано:

- добавлены уникальные ограничения для новых сущностей;
- `index-graph` импортирует новые labels идемпотентно;
- сохранена совместимость `check-graph` и `expand-chunks`.
- повторный `index-graph` теперь очищает старые keyword-derived `MENTIONS` и удаляет осиротевшие keyword-узлы, чтобы граф не расходился с новой классификацией сущностей.

### Шаг 3. Усилена диагностика импорта

Отчёт импорта теперь показывает не только число unresolved keywords, но и:

- самые частые unresolved keywords;
- распределение новых сущностей по классам;
- число удалённых устаревших keyword-derived `MENTIONS` и orphan-узлов при повторной синхронизации;
- понятную базу для следующей итерации ручной нормализации.

### Шаг 4. Подготовлен следующий переход ко второму уровню связей

После стабилизации новой разметки зафиксирован следующий ближайший фронт работ:

- `Article-[:PROPOSES]->Method`;
- `Method-[:HAS_ARCHITECTURE]->Architecture`;
- `Method-[:USES_TECHNIQUE]->Technique`;
- `Method-[:HAS_TUNING]->TuningStrategy`.

Это уже следующий этап после завершённого семантического обогащения первого уровня.
